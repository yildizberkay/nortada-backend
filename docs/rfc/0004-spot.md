## 1. Özet
Spot veri tabanı + geo nearby/search + spotların toplanması. Geo teknik: **sade Postgres** (bbox + haversine), PostGIS yok, 3rd-party geo API yok (D-003). Sourcing: OSM toplu tohum + shoreBearing türetme + Suggest Spot. Tam tasarım: [[spot-model-and-sourcing]].

## 2. Motivasyon / bağlam
Asıl problem: spot DB'miz yok. Temiz açık kaynak tek: OSM (ODbL). Beachhead Türkiye Ege (D-007), mimari global.

## 3. Kapsam (In / Out)
**In:** `spot` tablosu, nearby/search/detail endpoint'leri, OSM ingest pipeline (Trigger task), shoreBearing türetme, Suggest Spot (pending→publish), admin moderasyon uçları.
**Out:** hava/tahmin (RFC-0005 spot'a bağlanır), istasyon eşlemesi (D-004 gereksiz).

## 4. Veri modeli (Drizzle)
**`spot`** — `id`,`uid`,`name`,`country`,`region`,`locality`,`latitude`,`longitude`,`waterType` (sea|lake|bay|river|marina|open_coast),`supportedSports[]`,`skillSuitability`,`shoreBearingDeg`,`goodWindDirections[]`,`riskyWindDirections[]`,`hazards[]`,`source` (osm|curated|user_suggested),`osmId?`,`status` (published|pending|rejected),`createdBy?`,`createdAt`,`updatedAt`.
- Index: B-tree `(latitude, longitude)` (bbox ön-filtre); `status`; `osmId` unique (dedupe).
- Nearby: `WHERE lat BETWEEN ? AND ? AND lon BETWEEN ? AND ?` (radius→derece bbox) → haversine sıralama. 30k satırda ms-altı.

## 5. API yüzeyi
- `GET /v1/spots/nearby?lat=&lon=&sport=&radiusKm=` — yakın published spotlar.
- `GET /v1/spots/search?q=&sport=` — ada göre.
- `GET /v1/spots/:uid` — detay (hava RFC-0005'te birleşir).
- `POST /v1/spots/suggest` — kullanıcı önerisi → `status=pending`. Auth: user.
- Admin: `GET /v1/admin/spots?status=pending`, `PATCH /v1/admin/spots/:uid` (publish/reject/edit). Auth: developer/admin.

## 6. Servisler & mantık
- `SpotService` — nearby (bbox+haversine repo), search, detail, suggest, admin moderate + `requestOsmIngest` (task enqueue).
- `SpotIngestService` — OSM Overpass yanıtını normalize et (dedupe `osmId`), waterType/sports çıkar, `status=pending` yaz → admin publish. Ticari rehberler kopyalanmaz. **shoreBearing türetme (en yakın `natural=coastline` teğet açısı) → P1** (coastline geometrisi çekmek gerekir; küratör şimdilik elle set eder; [[../otonom-kararlar]] §23).
- Rüzgâr yönü → side/on/off-shore çevirimi `shoreBearingDeg` ile (`src/packages/geo` `windSide`; spot + hava kararında paylaşılan util). `shoreBearingDeg` = kıyıdan denize dış normal.

## 7. Arka plan işleri (Trigger.dev)
- `spot-osm-ingest` — bölge parametreli Overpass çekimi + normalize + shoreBearing (`schedules` veya manuel tetik). P1: dedupe/güncelleme diff cron.

## 8. Bağımlılıklar & entegrasyonlar
Overpass API (`https://overpass-api.de/api/interpreter`), ODbL atıf. Env: `OSM_OVERPASS_URL`. Coastline geometrisi için OSM `natural=coastline`.

## 9. Güvenlik & gizlilik
Suggest rate-limit. Admin uçları developer-only. OSM ODbL atıf UI'da.

## 10. Test
